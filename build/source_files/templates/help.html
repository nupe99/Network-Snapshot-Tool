How to Stop Application

Go to terminal windows where app was run and press CTRL-C

Taking this action will shutdown the net-snapshot application




How to Import Devices

There are two options to accomplish this:


Option #1 - Define Device Topology in a CSV file

In Net-Snapshot Web UI, select "Upload Device List" Tab

Downlaod sample CSV file and enter a row per-device.

OR

Option #2 - Manually define Device Topology in a YAML File

Place the Yaml topology file in the "net-snapshot/mount" directory. File must be named "topology.yaml"

Sample topology file can be found at "net-snapshot/mount/topology.yaml"




Steps to perform operational state capture (Snapshot)

1. Select "Collect" tab

2. Choose one or more operational state capture options in addition to one or more devices.

3. Select "Capture" button to begin capture


Note: Job ID is presented once the capture process begins. Once the capture completes, the job ID should be visible on the home page.



Steps to compare two operational state captures:

1. Select "Compare" tab

2. Select two job IDs from the list of captures

3. Select "Compare" button


Note: The the operational state changes between the two captures will be presented. If no changes are presented, no differences were detected between the two operational state captures.



Supported Operational State captures:

OSPF Interfaces - Capture state of OSPF enabled interfaces

OSPF SPF Runs - Capture the number of times OSPF SPF calculations have been performed.

OSPF Neighbors - Capture state of OSPF neighbor adjacencies

BGP Neighbors - Capture state of BGP neighbor adjacency changes

EIGRP Neighbors - Capture state of EIGRP neighbor adjacency changes

Route Table Changes - Capture state of IPV4 route tabling

HSRP - Capture state of HSRP

Packet Fragmentation Capture state of IP fragmentation (only IOS and IOS-XE devices supported)

Interfaces - Capture state of Interface counters. Interface up/down status, input errors, output drops, speed, duplex, interface mtu, and switchport mode (only applies to L2 interfaces).

DMVPN - For DMVPN hubs. Captures state of registered DMVPN spoke routers

Note: The the operational state changes between the two captures will be presented. If no changes are presented, no differences were detected between the two operational state captures.



See Github Repo README file for supported API calls
